# Hyprland config rules

These rules are instructions for Cursor when editing Hyprland configs.

## Allowed edit locations

Cursor may edit only files under:

- ~/dotfiles/hypr/.config/hypr/

For example:

- dotfiles/hypr/.config/hypr/hyprland.conf  
- dotfiles/hypr/.config/hypr/autostart.conf  
- dotfiles/hypr/.config/hypr/bindings.conf  
- dotfiles/hypr/.config/hypr/envs.conf  
- dotfiles/hypr/.config/hypr/input.conf  
- dotfiles/hypr/.config/hypr/looknfeel.conf  

Cursor must NOT edit:

- ~/.config/hypr/* directly  
- ~/.config/omarchy/current/theme/hyprland.conf  
- Anything in ~/.config/omarchy  
- Anything in ~/.local/share/omarchy  

## How Hypr config is applied

The repo version is applied to the system with:

```bash
omarchy-sync-hypr
```

Cursor should assume that:

- The repo is the source of truth  
- The live config under ~/.config/hypr is generated by omarchy-sync-hypr  

When describing workflows:

- Edit files in dotfiles/hypr/.config/hypr  
- Run dotfiles-sync push on the source device  
- On the target device run dotfiles-sync pull and omarchy-sync-hypr  

## Includes and structure

hyprland.conf may use source lines. A safe pattern is:

```ini
# Omarchy defaults, do not override these paths
source = ~/.local/share/omarchy/default/hypr/autostart.conf
source = ~/.local/share/omarchy/default/hypr/bindings/media.conf
source = ~/.local/share/omarchy/default/hypr/bindings/clipboard.conf
source = ~/.local/share/omarchy/default/hypr/bindings/tiling-v2.conf
source = ~/.local/share/omarchy/default/hypr/bindings/utilities.conf
source = ~/.local/share/omarchy/default/hypr/envs.conf
source = ~/.local/share/omarchy/default/hypr/looknfeel.conf
source = ~/.local/share/omarchy/default/hypr/input.conf
source = ~/.local/share/omarchy/default/hypr/windows.conf
source = ~/.config/omarchy/current/theme/hyprland.conf

# User overrides in ~/.config/hypr
source = ~/.config/hypr/monitors.conf
source = ~/.config/hypr/input.conf
source = ~/.config/hypr/bindings.conf
source = ~/.config/hypr/envs.conf
source = ~/.config/hypr/looknfeel.conf
source = ~/.config/hypr/autostart.conf
```

Cursor may adjust the contents of the user level files that live in the repo, but must never edit Omarchy default paths or their contents.

## Per device override support

User level Hypr config may include a local override file for per host tweaks, for example:

```ini
source = ~/.config/hypr/local.conf
```

Rules:

- local.conf must not be tracked in git  
- local.conf remains device specific  

Cursor must not create local.conf inside the repo. It can mention the pattern if asked.

## Keybinding style

When adding bindings:

- Keep a clear grouping by purpose  
- Use comments to label sections  
- Avoid conflicts with Omarchy default bindings unless intentional  

Example:

```ini
# Custom app shortcuts
bind = SUPER, Return, exec, kitty
bind = SUPER, B, exec, chromium
bind = SUPER, E, exec, thunar
```

Cursor should preserve existing style and structure in user files.

## Do not touch Omarchy theme Hypr config

Omarchy theme Hypr config lives in:

- ~/.config/omarchy/current/theme/hyprland.conf

Cursor must never edit that file and must never recommend editing it. All user customization goes into the user level files in dotfiles/hypr/.config/hypr.
